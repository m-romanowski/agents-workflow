# React JS

## Purpose

- reusable guidance for React component structure and review

## When To Load

- the task touches React component code, rendering structure, props/state boundaries, hooks usage, or frontend-specific review concerns

## Do Not Load By Default

- `structure.md` only when component structure or state/render boundaries matter
- `review.md` only when the task is a code review or review-style investigation

## Child Files

- `structure.md`: component responsibilities, composition, and state/render boundaries
- `review.md`: React-specific review cues and common misuse patterns

## Expected Project Overrides

- `instructions/project/index.md`
- `instructions/project/skill-profile.md`
- `instructions/project/modules/<module>.md` when local frontend structure differs

## Loading Examples

- component refactor: load `structure.md`
- React code review: load `review.md`
