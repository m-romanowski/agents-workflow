# Stack And Commands Template

## Purpose

- template for documenting verified stack choices and the commands used to run, build, test, and inspect the repository

## Template Rules

- use only verified commands from manifests, scripts, config, or operator-provided context
- separate required commands from optional or environment-specific ones
- keep command lists short and task-oriented
- mark unknown or unverified commands explicitly instead of guessing

## Suggested Sections

- runtime and language stack
- framework and tooling stack
- package-manager or build-tool notes
- run commands
- build commands
- validation commands

## Minimum Content Expectation

- enough verified command coverage for onboarding and safe task execution
