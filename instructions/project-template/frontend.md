# Frontend Template

## Purpose

- template for documenting frontend entry points, UI structure, and frontend-specific constraints within a repository

## Template Rules

- describe verified frontend structure only
- keep framework details secondary to boundaries, entry points, and responsibilities
- keep reusable technology-specific frontend guidance out of this file
- record uncertainty when frontend boundaries or runtime surfaces are not yet clear

## Suggested Sections

- frontend role in the repository
- entry points and runtime surfaces
- major architectural areas
- structural conventions
- extension points
- frontend-specific constraints

## Minimum Content Expectation

- enough information for the agent to understand where frontend changes belong, how the frontend is entered, and which local constraints matter
