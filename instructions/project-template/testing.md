# Testing Template

## Purpose

- template for documenting repository-specific testing setup, validation structure, and test-related constraints

## Template Rules

- describe actual test layout and commands, not generic testing advice
- distinguish unit, integration, end-to-end, or framework-specific layers only when verified
- keep reusable technology-specific testing guidance in `instructions/skills/`
- mark unknown validation flows explicitly

## Suggested Sections

- test layout
- test tooling
- validation commands
- environment or fixture assumptions
- known gaps or expensive suites

## Minimum Content Expectation

- enough information for the agent to choose the right validation surface and understand local testing constraints
