# Repo Map Template

## Purpose

- template for mapping the main repository areas, ownership boundaries, and source-versus-generated structure

## Template Rules

- describe only verified repository structure
- prefer paths and boundaries over prose
- mark generated, deployment-only, or fragile areas explicitly
- avoid inferring business meaning from directory names alone

## Suggested Sections

- repository shape
- main paths and responsibilities
- source versus generated outputs
- fragile or high-risk areas
- likely extension points

## Minimum Content Expectation

- enough structure for the agent to locate where changes likely belong without guessing
