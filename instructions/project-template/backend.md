# Backend Template

## Purpose

- template for documenting backend entry points, architectural areas, and backend-specific structural conventions

## Template Rules

- describe verified backend structure only
- keep framework details secondary to boundaries and responsibilities
- point to reusable skills only through project routing, not by duplicating skill guidance here
- record uncertainty when backend boundaries are not yet clear

## Suggested Sections

- backend role in the repository
- entry points
- major architectural areas
- structural conventions
- extension points
- backend-specific constraints

## Minimum Content Expectation

- enough information for the agent to understand where backend changes belong and what backend boundaries matter
